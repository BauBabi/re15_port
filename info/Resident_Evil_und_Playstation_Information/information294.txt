Additional info

First, you may ask what it is about those offsets. Well it's simple the first 8 bytes of the header are each 1 bytes values,
the following values from offset 8 and onwards are 4 byte long values (always unsigned = positive) and they point to the
individual data sections that are described below.

If you look at the camera (RID) offset decimal 36 or hex: 24 and read 4 bytes from it e.g decimal "100" or in hex "64 00 00 00" it would mean that you will
find the start of the RID camera data on decimal offset 100 / hex 64 00 00 00.

You might think the order of offsets given is at the same time the order of the data in the file, this is wrong!
Here is a list of the order data is actually stored in RDT files.

The optional means that this data is not required to have a basic room that will work without crashing.


If the header's ESP offset is pointing to 0, then ESP, EFF and TIM_EFF section will not be in the RDT.

08 - RVD (Camera switch zones)

The RVD data represent the switch triggers/zones the game uses to switch from one camera over to another one.
Each camera can have a total of 4 (seems to be the limit) switch zones, making a total of 64 switch zones in a room
that would have 16 cameras.

The first zone defined is not a switch, it rather defines the area in which 3D objects/models are visible in the current angle/camera.
Following that one a row of switches are placed. If a camera only has that first switch (maybe call it definition zone), then it is probably
only used in cutscenes.

The whole section consists of an array of 20 byte long RVD entries. Their structure looks like this:

12&#124; X3
14&#124; Z3
16&#124; X4
18&#124; Z4
Be_flag (2 Bytes)
----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
This should have the value 1, if you set it to 0 it will lead to the zone being non-active. So in short, it's a switch to (de-)activate the zone.

nFloor (2 Bytes)
----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
It most of the time is FF or 255 in decimal. By floor the heigth it will work on is meant. Setting it to 255 means it will work on all of the heights in the room
setting it to a specific floor will mean that it only works there.

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

11 - FLR (Floor/Footstep sound areas)

If the pointer to 18 - ESP is not 0 (meaning the room uses effect sprites) then there will be TIM files found from this offset onwards.

21 - TIM (MD1 textures)

If the Model value in the RDT header is not 0 (meaning the room uses MD1 3d models) then there will be TIM files used by the models, found from this offset and onwards until the end of the RDT file is reached.

They will use the same order as is given in the 10 - MD1/TIM Table section. There can be a maximum of 17 model entries in a room but the game never uses so much different objects. Instead the models often share textures.

After the last TIM file the end of the RDT file is reached.
